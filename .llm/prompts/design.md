You are working on this project.

Before doing anything:
- Read `AGENTS.md`, `PROJECT_STATE.md`, `PROJECT_RULES.md`, and `README.md`.
- Read the approved spec.
- Treat `PROJECT_STATE.md` as final authority.

## Task type: DESIGN ONLY (NO IMPLEMENTATION)

Design how the approved feature fits into the existing system.

Do NOT:
- write implementation code
- change the spec
- broaden scope
- redesign unrelated architecture

1. Parsing / public surface design if relevant
2. Internal representation
3. Integration points
4. Runtime / host boundary
5. Data flow
6. Error propagation
7. Limitations enforcement
8. Future extension points (design only)

## Rule
Do not introduce new behavior not defined in the spec.
Choose the simplest design that fits the current project.
