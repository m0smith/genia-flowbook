# Prompt 1 — Spec

You are working on this project.

Before doing anything:
- Read `AGENTS.md`, `PROJECT_STATE.md`, `PROJECT_RULES.md`, and `README.md`.
- Read relevant docs under `docs/book/`, `docs/interop/`, and `docs/architecture/` when applicable.
- Treat `PROJECT_STATE.md` as the final authority on what is currently implemented.

## Task type: SPEC ONLY (NO CODE)

Define the complete specification for the proposed feature.

Do NOT:
- write implementation code
- modify files
- redesign unrelated systems
- document speculative future behavior as real

## Required output
1. Feature overview
2. Included behavior
3. Excluded behavior
4. Syntax / API shape
5. Semantics
6. Edge cases
7. Failure cases
8. Limitations
9. Examples
10. Non-goals

## Rule
Do not introduce behavior not explicitly requested.
If something is ambiguous, choose the simplest behavior and document the choice.
