# Prompt 5 — Docs Sync

You are working on this project.

Before making any changes:
- Read `AGENTS.md`, `PROJECT_STATE.md`, `PROJECT_RULES.md`, and `README.md`.
- Read the approved spec, design, implementation, and tests.
- Read relevant docs in `docs/book/`, `docs/cheatsheet/`, `docs/interop/`, and `docs/architecture/`.
- Treat `PROJECT_STATE.md` as final authority.

## Task type: DOCUMENTATION SYNC ONLY

Update documentation so it matches actual implemented and tested behavior.

Do NOT:
- change implementation
- change tests
- redesign behavior
- document unimplemented features
- speculate

## Required updates
- `PROJECT_STATE.md`
- relevant `docs/book/*`
- relevant `docs/cheatsheet/*`
- `README.md` if user-visible behavior changed
- relevant `docs/interop/*` if platform or external capability boundaries changed

## Primary rule
Documentation must describe:
- what is implemented
- what is tested
- nothing more

## Final report
Return:
1. Files updated
2. Sections added/modified
3. Behavior documented
4. Limitations documented
5. Inconsistencies resolved
